## 一、tsconfig.json

如果一个目录下存在tsconfig.json，那么意味着这是一个ts项目的根目录，tsconfig.json文件指定了用来编译这个项目的根文件和编译选项。一个项目可以通过以下方式之一来编译：

- 不带任何输入文件的情况下调用tsc。编译器会从当前目录开始查找tsconfig.json,逐级向上级父目录查找。
- 不带任何输入文件的情况下调用tsc，且使用命令行参数--project或者-p 指定一个包含tsconfig.json的目录。

当命令行指定了输入文件时，tsconfig.json文件会被忽略。

```shell
yarn tsc --init
```

上面是这个默认文件的内容。

- target。 ts源码编译后的js版本
- module 当前代码的组织形式
- strict 开启js的严格模式
- esModuleInterop.允许从没有设置默认导出的模块中默认导入。不影响代码显示，只是为了类型检查。

## 二、TSLint

TSLint是ts代码的样式风格检查工具，类似于js的eslint。

```shell
yarn add tslint --dev

yarn tslint --init
```

会生成一个默认的tslint.json文件：

```json
{
    "defaultSeverity": "error",
    "extends": [
        "tslint:recommended"
    ],
    "jsRules": {},
    "rules": {},
    "rulesDirectory": []
}
```

## 四、husky

可以添加husky的钩子。

```json
#package.json
 "husky": {
        "hooks": {
            "pre-commit": "yarn tslint -c tslint.json './src/*.ts' './*.ts'"
        }
    }
```

这样每次提交代码。都会检查。

如果不能通过代码检查，则不允许提交。
